# 🔧 代理设置（解决内地「找不到服务器 / 云端同步失败」）

## 问题确认
手机浏览器打开 `https://crsuctkvtbprpphyel.supabase.com` 报「**找不到服务器**」（= `ERR_NAME_NOT_RESOLVED`），即 `*.supabase.co` 在内地被 DNS 污染，手机解析不出地址。代码和 Supabase 设置都已验证无误，唯一阻塞是网络层。

## 解法：Cloudflare Worker 免费中转（约 3 分钟）
Cloudflare 在内地可直连，它替手机去连 Supabase 再把结果回来。**免费、不用买服务器、不用装环境、不用绑卡。**

### 第 1 步：建免费 Cloudflare 账号
打开 https://dash.cloudflare.com → 用邮箱注册登录（免费，无需绑卡）。

### 第 2 步：新建 Worker 并粘贴代码
1. 左侧「Workers 和 Pages」→ 右上角「创建」→ 选「创建 Worker」。
2. Worker 名称随便填，如 `growth-garden-proxy`。
3. 把本文件夹里的 **`cloudflare-worker.js`** 内容**全部复制**，替换掉编辑器里原来的示例代码。
4. 点「部署」。

### 第 3 步：拿到你的代理地址
部署成功后，页面会显示地址，形如：
```
https://growth-garden-proxy.你的用户名.workers.dev
```
👉 这就是要填进 App 的「代理地址」（带 `https://` 整段复制）。

### 第 4 步：把地址发我
把第 3 步的 `https://xxxx.workers.dev` 地址**发给我**，我会直接写进 `index.html` 的 `SUPABASE_URL` 常量并重新发布。之后你打开花园网页即用，状态自动变绿 **☁️ 云端已连接·已同步**，无需任何手动配置，全家多设备实时同步。

---

## 如果 `*.workers.dev` 在你的网络也很慢/打不开（可选升级）
给 Worker 绑一个**你自己的域名**（任意域名，约 ¥10–30/年的 `.top`/`.cn` 即可），在 Cloudflare 里把域名「激活」（橙色云朵开启）后，把那个域名作为代理地址发我。Cloudflare 的域名在内地最稳。

## 完全不想用代理？（替代方案）
换内地原生、不用代理的后端 **LeanCloud**（已验证内地可直连），代价是改写 App 数据层（约半天）。需要的话告诉我。
