# 🔧 代理设置（解决内地手机「云端同步失败」）

## 为什么会失败？
App 的数据存在 **Supabase**（`*.supabase.co`）。这个域名在**内地网络被 DNS 污染**——手机根本解析不出地址，所以报「云端同步失败 / 已用本地数据」。**不是代码问题，也不是项目挂了**（项目状态一直是正常的）。

> 已验证：项目状态 `ACTIVE_HEALTHY`；只有 `*.supabase.co` 这个子域在内地打不开，`supabase.com` 主站正常。

## 最简解法：用 Cloudflare Worker 做免费中转（约 3 分钟，不用买服务器）
Cloudflare 在内地能直连，它替手机去连 Supabase，再把结果回来。**全程免费、不用装环境、不用绑卡。**

### 第 1 步：建一个免费的 Cloudflare 账号
打开 https://dash.cloudflare.com → 用邮箱注册登录（免费，无需绑卡）。

### 第 2 步：新建 Worker 并粘贴代码
1. 左侧点 **「Workers 和 Pages」** → 右上角 **「创建」** → 选 **「创建 Worker」**。
2. Worker 名称随便填，如 `growth-garden-proxy`。
3. 把本文件夹里的 **`cloudflare-worker.js`** 文件内容**全部复制**，替换掉编辑器里原来的示例代码。
4. 点 **「部署」**。

### 第 3 步：拿到你的「代理地址」
部署成功后，页面会显示你的地址，形如：
```
https://growth-garden-proxy.你的用户名.workers.dev
```
👉 **这就是要填进 App 的「代理地址」**（带 `https://` 整段复制）。

### 第 4 步：把代理地址写进 App（二选一）
App 里那个「⚙️ 重新配置后端」按钮已移除——代理地址以后**直接写进代码**，你不用手动填。

**方式 A（推荐，最省事）**：把第 3 步拿到的 `https://xxxx.workers.dev` 地址发给我，我直接写进 `index.html` 的 `SUPABASE_URL` 常量里。之后你打开网页即用，状态自动变绿 **☁️ 云端已连接·已同步**，无需任何配置。

**方式 B（自己来，不依赖我）**：在浏览器里清除本页的 localStorage（或换无痕窗口首次打开），页面会弹出「🌱 首次配置」框：
- **Project URL** → 粘贴 `https://xxxx.workers.dev`
- **anon public key** → 粘贴原来的 anon key（见 `index.html` 里的 `SUPABASE_ANON_KEY` 常量，或你 Supabase 项目 API 页的 anon key）
- 点「保存并进入」。状态应变绿 **☁️ 云端已连接·已同步**。

完成 🎉 之后换手机、给爸爸用，直接打开同一个网址即可，全家云端实时同步。

## 如果 `*.workers.dev` 在你的网络也很慢/打不开（可选升级）
给 Worker 绑一个**你自己的域名**（任意域名，够用一年约 ¥10–30 的 `.top`/`.cn` 即可），在 Cloudflare 里把域名「激活」（橙色云朵开启）后，把那个域名作为代理地址填进 App。Cloudflare 的域名在内地最稳。

## 完全不想折腾代理？（替代方案）
若你愿意换一个**内地原生、不用代理**的后端（数据存到国内服务器），可改用 **LeanCloud**（已验证内地可直连）。代价是需要把 App 的数据层改成 LeanCloud 的接口（约半天改造）。需要的话告诉我，我帮你改。
